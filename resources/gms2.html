GameMaker Studio 2 Guide

The fastest route to a playable 2D game. GML scripting is beginner-friendly, drag-and-drop is available as a stepping stone, and the IDE is purpose-built for game creation.

Beginner friendly 2D specialist Education licence available

When to Choose GameMaker

  • You're making a 2D game (platformer, top-down, puzzle, endless runner)
  • You want something playable on screen fast
  • You're new to programming — GML is the friendliest first language here
  • Your concept is scope-appropriate for the 12-week project
  • You like proof it can go big: Undertale, Hotline Miami and Chained Echoes shipped on GameMaker

Hardware Requirements

Minimum RAM4 GB
Recommended RAM8 GB
Storage (install)~3 GB
GPUAny modern GPU
OSWindows 10/11 or macOS 12+

Runs comfortably on every machine in the lab — the safe choice if you work across school and home computers.

Key File Types

  • .yyp — Project file (open this)
  • .yy — Individual resource files
  • .gml — GameMaker Language scripts
  • Sprites/sounds stored as .png/.wav inside the project

Sprite & Audio Import Specs

Sprites: import PNG files. GameMaker auto-detects sprite size. Set the Origin to the correct anchor point (usually centre for characters, top-left for tiles). Tile size must be consistent for tilesets.

Audio: import .wav or .ogg. For background music, tick Streaming. For SFX, leave as Compressed.

Getting Started

  1. Download GameMaker from gamemaker.io — ask your teacher for the Education licence key.
  2. Create a new project: File → New Project.
  3. Import your sprites: right-click Sprites → Create Sprite → Import.
  4. Create an Object for your player: right-click Objects → Create Object → assign your sprite.
  5. Build the official Fire Jump tutorial game (linked below) before starting your own project — it teaches Rooms, Events and GML in one sitting.
